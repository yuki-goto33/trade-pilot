# PoC-2: 銘柄フィルタリング条件の設計と universe 構築

機械的なフィルター条件だけで、スイング〜中長期投資に適した約 50 銘柄の universe を
**再現可能に**生成できるかを検証する。

## 実行方法

```bash
# 前提: リポジトリ直下の .env に JQUANTS_API_KEY を設定済み
source .venv/bin/activate
python poc/poc2_stock_universe/build_universe.py
```

- 初回実行: J-Quants API を約 21 回叩くため **約 5〜6 分**（無料プラン 5req/分 → 15.5 秒間隔）
- 2 回目以降: `data/poc2_cache/` のキャッシュを使うため **数秒**（API を叩かない）

## フィルター条件（最終形）

| # | 条件 | 値 | 根拠 |
|---|------|----|------|
| 1 | 市場区分 | 東証プライム | 機関投資家の参加が多く情報量・流動性が安定 |
| 2 | 商品区分 | 国内普通株のみ（`ProdCat == "011"`） | ETF / REIT / 出資証券 / 外国株を除外 |
| 3 | 種類株除外 | 5桁コード末尾が `0` | 優先株等の種類株（例: 伊藤園優先 25935）を除外 |
| 4 | 流動性下限 | 平均売買代金 **50億円/日 以上** | スイングで数百万円規模の売買をしても板に影響しない水準 |
| 5 | データ充足 | 直近20営業日中 **15日以上** 出来あり | 新規上場直後・長期売買停止銘柄を除外 |
| 6 | 選定順 | 平均売買代金の**降順**（同額はコード昇順） | 決定的な順序で再現性を担保 |
| 7 | セクター分散 | **33業種 1業種あたり 4 銘柄まで** | 銀行・電気機器などへの偏りを防止 |
| 8 | 目標銘柄数 | **50 銘柄** | PoC 計画の成功基準（50±10） |

パラメータは `build_universe.py` 冒頭の定数（`TARGET_SIZE` / `MIN_AVG_TURNOVER_JPY` /
`MAX_PER_SECTOR33` / `WINDOW_DAYS` / `MIN_PRESENT_DAYS`）で明示。

## データ取得の工夫

- **日足は `date=YYYYMMDD` 指定で全銘柄分を 1 リクエストで取得**（`/equities/bars/daily`）。
  銘柄コード別に回すと 1,571 銘柄 × 15.5 秒 ≈ 7 時間かかるところを、20 営業日分 = 約 21 リクエスト ≈ 5 分に短縮。
- 無料プランは **12 週遅延**のため基準日は「今日 − 12週」時点の最新営業日（銘柄マスタの `Date`）。
  銘柄マスタ・流動性測定の用途では遅延は実用上問題ない。
- レスポンスは `data/poc2_cache/`（gitignore 済み）にキャッシュし、再実行時は API を叩かない。
  祝日（空レスポンス）もキャッシュして再問い合わせを防ぐ。

## 出力

| ファイル | 内容 |
|---|---|
| `universe_50.json` | 機械可読。基準日・パラメータ・銘柄一覧（code / name / sector33 / 平均売買代金 / 規模区分） |
| `universe_50.md` | 目視レビュー用。33業種別に整理した一覧表 |

時価総額は J-Quants 無料プランの銘柄マスタ・日足に含まれないため未収載
（`ScaleCat`（TOPIX 規模区分: Core30 / Large70 / Mid400 / Small）を代替情報として収載）。

## 結果サマリー（2026-07-04 実行・基準日 2026-04-10）

- マスタ 4,450 銘柄 → プライム国内普通株 **1,571 銘柄** → 流動性フィルター（平均売買代金 50億円/日以上・20営業日中15日以上出来）通過 **211 銘柄** → セクター分散制約適用後 **50 銘柄**
- 算出窓: 2026-03-13〜2026-04-10 の 20 営業日（2026-03-20 春分の日は API が空レスポンス → 自動スキップ）
- 初回実行 365 秒（API 21 リクエスト）/ 2回目 0 秒（全キャッシュ）

## 再現性の確認

同一パラメータで 2 回実行し、`universe_50.json` / `universe_50.md` を `diff` で比較して
**バイト単位で同一**であることを確認済み（2回目は全キャッシュのため 0 秒で完了）。
選定ロジックは決定的ソート（平均売買代金 降順 → コード昇順）のみで乱数・時刻依存なし。

## 気づき・課題

- **20営業日の平均売買代金は直近のテーマ株急騰の影響を受けやすい。** 例: 日東紡績
  （TOPIX Small 1 ながら平均 398億円/日で選定）、三井海洋開発など。明らかに不適切とまでは
  言えない（実際に流動性は十分）が、スイング〜中長期の対象としては入れ替わりが激しくなる懸念。
  対策候補: 算出窓を 60 営業日に延長 / 平均→中央値 / `ScaleCat`（TOPIX Mid400 以上）の追加条件。
- **セクター上限が「業種内の巨大流動性銘柄」を優先し、日立・ソニー級を弾く。** 半導体相場の
  局面では電気機器の枠が半導体関連で埋まる。業種上限を 5〜6 に緩めるか、17業種（`S17`）での
  制約に切り替える選択肢がある。
- 時価総額は無料プランのマスタ・日足に含まれず、フィルターに使えなかった
  （売買代金 + `ScaleCat` で実用上は代替可能）。
- 銘柄マスタ・日足とも 1 日分は 1 リクエスト・ページネーションなしで取得できた
  （4,450 行 ≈ 1.1MB）。レート制限 5req/分が実質唯一のボトルネック。
