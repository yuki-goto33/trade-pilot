# PoC-2 残検証: 50銘柄 universe のデータ取得可否チェック

`universe_50.json` の50銘柄全銘柄について、PoC-1 で検証済みの各データソースから
欠損なくデータを取得できるかを実走で確認した。

- 実施日: 2026-07-04（金）10:05〜10:11 JST
- universe: `poc/poc2_stock_universe/universe_50.json`（50銘柄、英字入りコード 285A を含む）
- 実行方法: PoC-1 スクリプトを universe 差し替え対応に一般化した上で
  `UNIVERSE_FILE=poc/poc2_stock_universe/universe_50.json` を指定して逐次実行

### 朝バッチ相当の実測所要時間（50銘柄）

株価（yfinance）+ 銘柄ニュース（Google News）+ 開示（TDnet）の逐次実行で
**合計 318.7 秒 ≈ 5分19秒**。11銘柄時の実測（2.7s + 27.3s + 38.2s = 68.2s）から
ほぼリクエスト数比例で外挿どおり。**7:00 台に開始すれば 8:30 の判断タイムリミットに余裕で間に合う**。
内訳:

| ステップ | 11銘柄実測 | 50銘柄実測 | 支配要因 |
|---|---:|---:|---|
| yfinance 日足30日 | 2.7s | 15.7s | バッチDL（threads=False 逐次） |
| Google News RSS | 27.3s | 122.9s | リクエスト間 2.0s × 50 |
| yanoshin TDnet | 38.2s | 180.1s | リクエスト間 3.5s × 50 |
| 合計 | 68.2s | **318.7s** | |

※ J-Quants 日足は無料プランが12週遅延のため朝バッチの当日判断には使わない前提
（履歴・バックテスト用）。銘柄別に叩くと 50銘柄 × 15.5s ≈ 13分だが、
PoC-2 で確認済みのとおり **`date=YYYYMMDD` 指定なら1営業日 = 1リクエスト**で
全銘柄分取れるため、日次更新は 15.5s/日で済む。

## 個別の気づき（欠損ではないが記録）

- **8725 MS&AD: ニュース 0件。** フィードは正常応答（HTTP 200）で「直近1日にヒットなし」。
  検索クエリは NFKC 正規化した `"MS&ADインシュアランスグループホールディングス" OR "8725"` で、
  正式名称が長すぎて表記揺れに弱い可能性がある。universe に `news_name`（例: `MS&AD`）を
  明示できる仕組みは実装済みなので、要件定義時に通称マップの整備を検討。
- **9434 ソフトバンク: ニュース 100件（上限張り付き）。** クエリ `"ソフトバンク"` が
  ソフトバンクグループ（9984）や「ソフトバンクホークス」等の記事も拾っている可能性が高い。
  件数が多い分には欠損ではないが、銘柄ニュースとしての精度は要件定義でのフィルター課題。
- **285A キオクシア: TDnet 38件（他銘柄は limit=50 に到達）。** 2024年12月上場で
  開示の総数自体が少ないため。取得失敗ではない。英字入りコードも
  yfinance（`285A.T`）・TDnet（`285A`）・J-Quants（`285A0`）すべて問題なし。
- 銘柄名の全角英数（ＫＤＤＩ・ＮＴＴ・ＨＯＹＡ 等 8銘柄）は universe ローダーが
  NFKC 正規化してニュース検索に使用しており、いずれも正常に件数を取得できた。

## J-Quants キャッシュ検証の方法

API を叩き直さず、PoC-2 の `data/poc2_cache/bars_YYYYMMDD.json`（21ファイル、
うち 2026-03-20 春分の日は空 = 祝日キャッシュ）を走査し、50銘柄の5桁コード
（`code5`）の出現日数と OHLCV の null を確認した。
結果: **全50銘柄が 20/20 営業日に出現、OHLCV/Volume の null は 0 行**。

## スクリプトの一般化（universe 差し替え対応）

PoC-1 の fetch スクリプト群は11銘柄固定だったため、以下のとおり一般化した
（デフォルトは従来の11銘柄で後方互換維持。`run_all.py` は変更不要）。

| ファイル | 変更内容 |
|---|---|
| `poc/poc1_data_sources/universe.py` | `load_universe(path)` を追加。優先順位: 引数 → 環境変数 `UNIVERSE_FILE` → デフォルト11銘柄。`universe_50.json` 形式（`{"stocks": [...]}`）とリスト直接の両対応。全角英数名は NFKC 正規化して `news_name` を自動補完。`yf_tickers()` 等は universe 引数を受け取れるよう拡張 |
| `poc/poc1_data_sources/common.py` | `parse_universe_arg()`（共通 `--universe` オプション）を追加 |
| `poc/poc1_data_sources/fetch_prices_yfinance.py` | `main(universe_path)` 化 + `--universe` 対応 |
| `poc/poc1_data_sources/fetch_news_google.py` | 同上 |
| `poc/poc1_data_sources/fetch_disclosures_yanoshin.py` | 同上 |
| `poc/poc1_data_sources/fetch_prices_jquants.py` | 同上（今回は実行せずキャッシュで代替） |
| `poc/poc1_data_sources/fetch_financials_edinet.py` | 同上（今回の検証対象外） |

実行例:

```bash
# 引数指定
.venv/bin/python poc/poc1_data_sources/fetch_prices_yfinance.py \
  --universe poc/poc2_stock_universe/universe_50.json

# 環境変数指定（run_all.py 経由でも有効）
UNIVERSE_FILE=poc/poc2_stock_universe/universe_50.json \
  .venv/bin/python poc/poc1_data_sources/fetch_news_google.py
```
